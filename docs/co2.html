面试总结分析

前言 #

最近面试了多家公司，对面试的过程做了一下分析总结。希望对正在找工作或者准备找工作的同学一点点帮助。如果有什么问题可以联系作者--王小壮壮

1.个人介绍 #

常规自我介绍+毕业时间+所呆过的公司+所担任的职责

2.说一说最近做的几个项目 #

描述一下你觉得最深刻的一个项目（项目背景+需求+功能实现+采用的架构）,这里有个重要的点就是在叙述项目的时候，先把功能讲清楚，然后再说具体的功能使用什么技术去实现，实现过程中是否存在难点，是怎么解决的？面试官主要关系你在其中做了什么，在做的过程中有没有什么技术难点或者技术亮点，你又是怎么去解决处理的。

  • 1.相关知识点 mq（在什么场景下使用的，为什么要使用，如何保证消息的一致性？消息的持久性。幂等性等）
  • 2.redis（为什么要使用redis 有什么好处 你觉得redis为什么这么快（内存+单线程+多路复用IO-epoll模型）,redis常用的数据类型，持久化机制,淘汰策略，有哪些优缺点 你是如何设计这些缓存机制的）

3.对技术的方面的提问 #

多线程 #

（在此项目中你是如何使用的 有什么好处 在处理问题中有没有遇到什么并发问题 是如何去处理的

线程池 #

既然用到了多线程 那肯定是用到线程池

  • 一般是怎么创建线程池的
  • 线程池的基本参数
  • 丢弃策略
  • 提交方式
  • 怎么设置其中的核心线程数，最大线程数
  • excute、submit有什么区别
  • 创建线程的几种方式
  • 怎么接收返回的结果

锁 #

  • synchronized（可重入）
    • 底层的实现
    • 锁的升级过程（偏向锁 轻量级锁 重量级锁）
    • synchronized(this)代表的意思
  • reentrantLock （可重入）
    • 公平锁 底层实现原理
    • 非公平锁 底层实现原理
  • 以上两个的区别
  • 怎么实现一个线程的精准唤醒 condition+Lock
  • CountDownLatch
    • 用法
  • **CyclicBarrier **
    • 如何实现
  • Semaphore
    • 平常是如何使用的
  • ThreadLocal 是如何实现的 解决什么问题
  • CurrentHashma 底层实现

分布式锁 #

有使用过分布式锁吗？是用什么实现的？redis？还是其他的什么 首先分布式锁要解决的问题就是分布式环境下同一资源被多个进程进行访问和操作的问题,既然是同一资源,那么肯定要考虑数据安全问题.其实和单进程下加锁解锁的原理是一样的,单进程下需要考虑多线程对同一变量进行访问和修改问题,为了保证同一变量不被多个线程同时访问,按照顺序对变量进行修改,就要在访问变量时进行加锁,这个加锁可以是重量级锁,也可以是基于cas的乐观锁.
